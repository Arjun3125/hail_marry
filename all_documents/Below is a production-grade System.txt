# System Overview

**Project Name:** VidyaOS – AI Infrastructure for Educational Institutions
**Version:** v0.1 (Pilot Architecture)
**Deployment Model:** Hybrid (Cloud ERP + Local AI Compute Node)
**Target Stage:** 2–3 Pilot Schools

---

# 1. Executive Summary

VidyaOS is a multi-tenant educational AI infrastructure platform that combines:

* Institutional ERP capabilities
* AI-powered document intelligence
* Student performance-aware tutoring
* Secure role-based access
* Scalable microservice architecture

The system is designed for bootstrap deployment using a local GPU inference server while keeping the core SaaS ERP layer cloud-hosted.

This architecture enables:

* Low operational cost during pilot
* Full control over AI inference
* Clear upgrade path to cloud GPU cluster

---

# 2. Architectural Philosophy

## Design Principles

1. Multi-tenant by default
2. Strict separation of operational data and AI data
3. AI as augmentation, not authority
4. Cloud-ready architecture from day one
5. Security-first design
6. Observability and metrics-driven iteration

# 4. Core System Components

## 4.1 API Gateway (Cloud VPS)

Responsibilities:

* TLS termination
* JWT validation
* Role-based access control
* Tenant routing
* Rate limiting
* Usage logging

Tech stack:

* Nginx
* FastAPI
* Redis (for caching and throttling)

---

## 4.2 Core ERP Service

Handles deterministic institutional operations:

### Modules

* Student Management
* Attendance Tracking
* Marks & Results
* Assignment Tracker
* Timetable Engine
* Complaint Portal
* Lecture Library Management

Database:

* PostgreSQL
* Strict `tenant_id` isolation

---

## 4.3 AI Engine (Local GPU Server)

Runs on custom-built server with RTX 4090 (recommended).

### Subsystems

#### 4.3.1 Ingestion Service

* PDF parsing
* Transcript ingestion
* Document chunking (hierarchical)
* Metadata extraction

#### 4.3.2 Embedding Service

* Uses local embedding model
* Stores vectors in tenant-isolated namespaces

#### 4.3.3 Retrieval Layer

* Top-K semantic search
* Optional reranking
* Deduplication
* Context compression

#### 4.3.4 LLM Inference Layer

* Quantized 14B model (or equivalent)
* Controlled context window
* Structured prompting
* Citation-enforced outputs

#### 4.3.5 Task Template Engine

Supports:

* Q&A
* Study Guide
* Quiz Generation
* Topic Summary
* Concept Breakdown
* Weak-topic remediation

---

# 5. Multi-Tenant Model

Each institution has:

* Unique tenant_id
* Isolated Postgres rows
* Separate vector namespace
* Independent usage tracking
* Configurable AI limits

No cross-tenant data mixing.

---

# 6. Data Flow

## Student AI Query Flow

1. Student submits question
2. API validates JWT + tenant
3. Request routed to AI microservice
4. Retrieval from tenant vector DB
5. Reranking + context assembly
6. LLM inference
7. Citation injection
8. Response returned
9. Usage logged for billing

# 12. Key Risks

| Risk             | Mitigation                        |
| ---------------- | --------------------------------- |
| Power outage     | UPS + auto restart                |
| Hardware failure | Daily backup + offsite copy       |
| Latency spike    | Request queueing                  |
| Overload         | Tenant-level rate limiting        |
| Data breach      | Strict isolation + firewall rules |

# 14. Strategic Position

VidyaOS is not an ERP with AI features.

It is:

> A verticalized, multi-tenant AI infrastructure layer for institutions, with ERP as the interface shell.

The long-term moat lies in:

* Institutional data accumulation
* Personalized AI learning loops
* Multi-year student performance memory
* Proprietary evaluation benchmarks
