# Admin Review Dashboard

**Project:** VidyaOS – AI Infrastructure for Educational Institutions
**Version:** v0.1 (Pilot → Scale)
**Role:** Tenant Admin (School-Level)
**Access Level:** Highest within Tenant

---

# 1. Dashboard Purpose

The Admin Review Dashboard must allow:

1. Visibility into institutional performance
2. Monitoring AI usage & cost
3. Reviewing system health
4. Detecting abuse or anomalies
5. Managing user roles
6. Maintaining governance control

Admin must feel in control.

# 13. Filtering Controls

Admin dashboard must allow filtering by:

* Date range
* Academic year
* Class
* Subject
* Role

All queries must enforce tenant_id.

---

# 14. Data Refresh Strategy

Real-time not required everywhere.

Recommended:

* KPI refresh: every 60 seconds
* AI usage: every 5 minutes
* Academic stats: daily recalculation
* Reports: on-demand

---

# 15. Performance Optimization

Admin dashboard queries can be heavy.

Use:

* Precomputed aggregation tables
* Cached statistics
* Background cron jobs

Avoid real-time heavy joins.

---

# 16. UX Design Principles

Admin UI must feel:

* Institutional
* Structured
* Clean
* Data-dense but readable
* Not chat-focused

Color coding:

* Green: Healthy
